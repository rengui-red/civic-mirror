\# 技术结构蓝图 · 预约文明之境 v1.0



\## 设计原则



1\. \*\*协议驱动\*\*：一切对话与行为受显式的“文明协议”约束，而非黑盒模型。

2\. \*\*数据闭环\*\*：人与AI的交互自动生成训练数据，反哺系统进化。

3\. \*\*分级解锁\*\*：MVP → 核心平台 → 开放生态，逐步扩充模块。

4\. \*\*可审计性\*\*：治理相关数据写入不可篡改日志，支撑“文明陪审团”透明运作。

5\. \*\*隐私优先\*\*：用户画像默认端侧加密存储，不上传服务器。

6\. \*\*防依赖\*\*：不设推送、打卡、等级体系，内置关怀式熔断。

\## 各层详细定义



\### 核心层 (Core Layer) —— 系统的“灵魂”



| 模块 | 职责 | 关键技术 |

|:---|:---|:---|

| 文明协议解释器 | 将文明对话协议转化为AI可执行的状态机。强制执行四步引导，禁止直接给答案。 | 状态机引擎；协议文件解析；与仁格映照引擎双向通信 |

| 仁格映照引擎 | 承担“映照”逻辑。分析用户的价值排序、共情半径、视角移动，生成观察性描述而非评判。 | 价值关键词聚类；情感归因；观点状态追踪；禁止评判词过滤 |

| 记忆与动态画像 | 存储用户的价值偏好、认知模式、情绪触发点。\*\*仅存于端侧加密区，不上传\*\*。 | IndexedDB + 端到端加密 + 差分隐私 |



\### 服务层 (Service Layer) —— 业务逻辑中枢



| 模块 | 职责 | 依赖的核心模块 |

|:---|:---|:---|

| 场景引擎 | 基于参数矩阵动态生成道德困境场景，管理分支叙事与后果呈现。 | 协议解释器 |

| 映照分析引擎 | 在用户行动时实时分析情绪、价值、视角，生成映照数据。 | 仁格映照引擎 + 记忆画像 |

| 报告生成器 | 将分析数据转化为结构化的“文明映照报告”。 | 映照分析引擎 |

| 人格守护模块 | 防依赖熔断、反诱导提醒、数据主权管理。 | 记忆画像 |



\### 应用层 (Application Layer) —— 用户体验



\*\*已实现 (v0.3)\*\*：

\- 场景广场：展示可用场景，用户选择进入

\- 沉浸抉择：场景叙事 + 分支抉择 + 后果呈现

\- 反思对话：抉择后的开放式反思对话

\- 映照报告：基于选择与对话生成的观察性报告



\*\*规划中\*\*：

\- 成长档案：多场景报告的时间线回顾

\- 共情地图：基于多场景数据的共情半径可视化

\- 换位挑战：强制分配对立角色进行视角练习

\- AI试炼沙盒：供AI进行伦理训练的控制台



\### 治理层 (Governance Layer) —— 文明的自我进化



| 模块 | 职责 |

|:---|:---|

| 文明陪审团审议系统 | 争议案例提交、多方投票、判例生成 |

| 协议演化管理器 | 基于判例生成新版本协议提案、测试与发布 |

| 不可篡改审计日志 | 记录协议变更、陪审团决议、重大伦理事件 |

| 开放治理协议 | 允许外部社区分叉或贡献文明规则模块 |



\## 三大数据流轴



\*\*人类数据流\*\*：

用户交互 → 应用层 → 映照分析引擎 → 记忆画像(本地加密) → 报告生成器 → 用户反馈



\*\*AI数据流\*\*：

场景引擎生成困境 → 用户抉择数据 → 文明共演数据集(脱敏) → 用于AI伦理模型微调



\*\*治理数据流\*\*：

人机交互争议触发点 → 提交陪审团 → 审议讨论 → 生成判例 → 协议演化 → 更新至核心层



\## 隐私设计



\- 用户画像、对话记录、报告：\*\*默认全部存储在客户端IndexedDB\*\*

\- 加密方案：端到端加密，用户可随时导出或一键擦除

\- 服务器端：仅存储脱敏统计级数据，且需用户主动授权

\- 未来探索：端侧模型推理 + 零知识证明
